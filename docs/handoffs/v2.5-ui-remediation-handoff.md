# v2.5 UI 修正引き継ぎ資料

最終更新: 2026-05-30

## 1. 目的

現行実装の機能を維持しながら、`docs/ui-reference/` の参考画像に沿って UI を修正する。
対象は **v2.5 相当**。汎用カード中心の現行 UI を、画面ごとの情報設計を持つ UI へ段階的に変更する。

本資料は UI 修正 PR の実装指示書である。Firestore の実データ保存と通知メールの実配信は別 PR とする。

## 4. 現状分析

### 4.1 技術構成

- Next.js 15 App Router / React 19 / TypeScript / Tailwind CSS
- Repository 抽象を経由した LocalStorage 保存
- Firebase 匿名認証は利用可能。Firestore repository は LocalStorage に委譲するスタブ
- Vitest / Testing Library / Playwright
- Redux 等のグローバルストアは未使用。Client Component の `useState` / `useEffect` を利用
- EC は `merchantId: string` と merchants マスタで管理

### 4.2 UI が参考画像と異なる主因

- `tailwind.config.ts` と `src/app/globals.css` がクリーム背景・緑 CTA を定義している。
- `src/components/ui/Card.tsx` と `src/components/ui/Button.tsx` が汎用的で、参考 UI の
  青・オレンジ基調、角丸、情報階層を表現していない。
- 各画面が汎用カードを並べる構成に寄っており、参考 UI の画面固有レイアウトを実装していない。
- PC とスマートフォンで必要な構造差が十分に分離されていない。

## 5. UI 修正対象

### 5.1 全体テーマと共通 UI

参照: `docs/ui-reference/01_home_responsive.png` 〜 `07_article_monthly_sale_strategy_responsive.png`

- メインカラーを青、主要 CTA をオレンジ、補助背景を薄青に寄せる。
- 楽天は赤、成功状態は緑を使用する。Yahoo はテキストバッジで識別できる配色を用意する。
- カード角丸はおおむね 16〜20px、ボタン角丸は 12〜16px を基準にする。
- 余白、見出しサイズ、カード内の情報階層を共通化する。
- Amazon、楽天、Yahoo の公式ロゴ画像は使用しない。EC 識別はテキストバッジと配色で表現する。

主な対象:

- `tailwind.config.ts`
- `src/app/globals.css`
- `src/components/ui/Card.tsx`
- `src/components/ui/Button.tsx`
- `src/components/layout/SiteHeader.tsx`
- `src/components/layout/SiteFooter.tsx`

### 5.2 ヘッダー、フッター、モバイルナビ

参照: `docs/ui-reference/01_home_responsive.png`

- PC はブランド名、主要ナビ、検索または検索導線を整理する。
- スマートフォンは上部ヘッダーと下部ナビを分離する。
- 下部ナビには目立つ登録導線を配置する。
- 現在地が分かるアクティブ表示を追加する。
- 記事詳細の固定シェアバーとモバイル下部ナビが重ならないようにする。

### 5.3 トップ画面

## 6. UI 修正と同時に行う小規模な安全対策

楽天 API の通常経路では画像ホスト検証済みだが、保存値が改ざんされた場合の防御を追加する。

- 楽天画像 URL の許可判定を共通関数へ集約する。
- LocalStorage から読み込む際と描画する際にも許可ホストを再検証する。
- 許可外 URL は表示しない。外部画像へフォールバックしない。
- 任意の画像 URL 入力欄は追加しない。

主な対象:

- `src/lib/product-search/rakuten.ts`
- `src/lib/utils/wish-item.ts`
- `src/features/wishlist/WishlistList.tsx`
- 関連テスト

## 7. 今回変更しないもの

- Firestore の実保存、同期、LocalStorage からのデータ移行
- 復旧コードによる実際の復旧処理
- 通知メールの実配信、配信失敗処理
- Repository interface の大規模変更
- `merchantId: string` と merchants マスタ方式
- 楽天公式 API 経由の商品検索
- 既存のアフィリエイト URL 変換
- 共有時のプライバシー保護
- 法務ページ
- スクレイピング、任意画像 URL 入力、外部 EC 画像の無断取得

## 8. 実装時の注意点

- 大規模な全面書き換えを避け、共通 UI → 各画面の順に段階的に修正する。
- 既存の買い時スコア、価格計算、履歴、楽天 API 画像、通知設定 UI を壊さない。
- 既存の LocalStorage データを読める状態を維持する。
- `docs/ui-reference/` はレビュー専用であり、アプリから配信しない。
- `.env.local` や API キーをコミットしない。
- Claude Code と Codex を同時に同じブランチ・同じファイルへ編集させない。

## 9. 実装順序

1. 作業環境と Git 差分を確認する。
2. 共通テーマ、Button、Card、ヘッダー、フッター、モバイルナビを修正する。
3. トップ、カレンダー、セール詳細を修正する。
4. 欲しいもの登録、一覧を修正する。
5. 履歴、記事詳細、必要な状態 UI を修正する。
6. 復旧コード画面を修正する。
7. 楽天画像 URL の再検証を追加する。
8. テストとビルドを実行する。
9. PC 幅約 1200px とスマートフォン幅約 390px で全参考画面を確認する。
10. 参考 UI と意図的に異なる点、未対応点を報告する。

## 10. 受け入れ基準

- `docs/ui-reference/01_*.png` 〜 `07_*.png` の各画面について、主要構造が反映されている。
- 青・オレンジ基調、薄青背景、カード角丸、CTA の優先順位が統一されている。
- カレンダー初期表示月が現在月である。
- Amazon、楽天、Yahoo のフィルタが UI に表示される。
- PC のカレンダーに選択日詳細パネル、スマートフォンにボトムシートがある。
- 欲しいもの登録は PC で画像左・フォーム右、スマートフォンで画像先行になっている。
- 実質価格メモが基本入力として見える。
- モバイル下部ナビと記事共有バーが重ならない。
- 復旧コード画面は未提供の実処理を成功したように見せない。
- 楽天 API 画像以外の任意外部画像を表示しない。
- 「最安値」「最安保証」等の断定表現や公式ロゴを追加していない。
- `npm run lint`、`npm run test`、`npm run test:e2e`、`npm run build` が成功する。
