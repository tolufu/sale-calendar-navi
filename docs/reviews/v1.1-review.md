## 必須修正（リリースブロッカー）

### ① 初期の参考リンク行で「保存する」が必ず失敗する 🛑
- 現象: フォーム初期状態の参考リンクは
  `{ kind:"kakaku", label:"価格比較メモ", url:"" }` 1行（`WishlistForm.tsx:71-73`）。
  ユーザーがこの行のURLを入力しない／ラベルを消さない限り（＝**詳細入力を開かない通常フローを含む**）、
  保存時に `parseReferenceLinks` が走る。
- 原因: フィルタが `draft.label.trim() || draft.url.trim()`（OR）のため、**ラベルだけある空URL行が残り**、
  続く `new URL(draft.url)` が `new URL("")` で `TypeError: Invalid URL` を送出（`WishlistForm.tsx:37-40`）。
  例外は `handleSubmit` の `catch` に落ち、ユーザーには `"Invalid URL"` という不可解なメッセージが出て保存できない。
- 影響: **欲しいもの登録の基本フローが成立しない**（テストはフォーム送信を踏まないため検知漏れ）。
- 修正方針（いずれか）:
  - (a) 初期行を `url` 必須前提に変える：保存対象は「URLが入っている行」のみとし、URLが空の行は破棄する
    （設計 §8.3「空行は保存対象外」に合致）。`new URL()` は try/catch で囲み、失敗時は日本語の検証エラーにする。
  - (b) 初期行のラベルを空にする（`label:""`）。こうすれば未入力行は OR フィルタからも漏れて破棄される。
  - 推奨は (a)。「URLあり＆ラベル空」のときだけ「表示名を入力してください」を出す順序にする。

## 推奨修正（v1.1 内で対応が望ましい）

### A. 参考リンクの不正URLが生エラーで表示される
- `parseReferenceLinks`（`WishlistForm.tsx:36-54`）は `new URL()` を try/catch せず呼ぶため、
  不正URL入力時に `"Invalid URL"` がそのまま露出。①の修正と合わせ、行ごとに日本語の検証メッセージ
  （「参考リンクは http/https のURLで入力してください」「表示名を入力してください」）へ集約する。

### B. 件数上限が未実装（設計 §8.3「最大5件目安」）
- 「追加」ボタンに上限がなく無制限に行追加できる。5件で「追加」を無効化、もしくは保存時に上限チェックを入れる。

### C. 外部リンクの `rel` を `noopener noreferrer` に統一
- レビュー観点4は `rel="noopener noreferrer"` を要求。現状は全外部リンクが `rel="noreferrer"`
  （`WishlistList.tsx:208,223,228`）。`noreferrer` は現代ブラウザで noopener を含意するため**安全性に実害はない**が、
  チェックリスト準拠のため明示的に `noopener noreferrer` へ揃えることを推奨。

### D. テスト追加
- 参考リンク検証（不正プロトコル拒否・ラベル空・件数上限）の単体テストがない（①の回帰防止に必須級）。
- 小数の `productPrice` と `pointRate` の丸め（`Math.round`）・0クランプの内訳経由テストがない。
- フォーム送信で内訳・候補・参考リンクが永続化される最小コンポーネントテストがあると①の再発を防げる。

### E. `desiredPrice` 保存時の負値バリデーション
- `desiredPrice: desiredPrice ? Number(desiredPrice) : null`（`WishlistForm.tsx:164`、一覧の `saveEdit` も同様）。
  `min="0"` は入力ヒントに過ぎず、負値が保存され得る（次回読込の `toNumberOrNull` で null に落ちるため不整合）。
  保存前に `< 0` を弾くか 0 にクランプする。

### F.（画像比較）EC選択UIを自動判別前提に寄せる
- 参照フォームに EC セレクトはなく「URLから自動判別」を明記。実装は可視セレクトを残している。
  自動判別結果のラベル表示＋（必要時のみ）手動上書き、に整理すると参照に近づく。要件確認のうえ判断。

### G.（画像比較）「現在の実質価格メモ」を折りたたみ外へ
- 参照は希望価格直下に常時表示。実装は詳細折りたたみ内（`WishlistForm.tsx:257-260`）にあり、開かないと見えない。
  メモは主要項目として折りたたみ外（常時表示）へ移すと情報順序が参照と一致する。

### H.（画像比較）各項目の補助説明文を追加
- 参照は各フィールドに説明文（通知条件・自動判別・実質価格の計算式など）を添える。
  実装の補助文は薄め。少なくとも計算式と「自動判別」の一文をフィールド近傍に出すと親切。

### I.（画像比較・設計§6）「希望価格内」中立バッジが未実装
- 設計§6は主候補/最小の実質価格が `desiredPrice` 以下なら「希望価格内」、超過なら「希望価格まで {¥差}」の
  中立バッジを規定。`WishlistList.tsx` に該当表示がない。断定語を避けた中立表現で実装する。
