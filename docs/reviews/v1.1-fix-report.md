## 対応内容

- 初期表示の参考リンク行 `{ label: "価格比較メモ", url: "" }` が保存時に `new URL("")` で失敗する問題を修正。
  - 参考リンク保存ロジックを `src/lib/utils/reference-link.ts` に純関数として切り出し。
  - URLが空の行はラベル有無に関わらず保存対象外。
  - URLあり・表示名なしは「参考リンクの表示名を入力してください。」を表示。
  - 不正URL・不正プロトコルは「参考リンクは http または https のURLで入力してください。」を表示。
  - 保存件数は5件まで。追加ボタンも5件で無効化。
- 欲しいもの登録・一覧編集の `desiredPrice` は、空欄または負値・不正数値を `null` として保存するよう修正。
- 一覧の外部リンク `rel` を `noopener noreferrer` に統一。
- テストを追加・補強。
  - 参考リンク検証: 空URL行スキップ、http/https許可、不正プロトコル拒否、表示名必須、5件上限。
  - 実質価格計算: 小数と `pointRate=0.5` の `Math.round`、0クランプ。
  - 楽天AF変換: 既存 `scid` を上書きしないこと。
  - データ移行: 負の希望価格と不正な参考リンクを正規化時に破棄すること。
