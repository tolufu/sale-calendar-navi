## 2. v1.2 特有の落とし穴（必ず伝える）

1. **実メール送信を本番運用前提で実装しない**
   通知設定は「保存」と「リマインダー候補のローカル生成」まで。SMTP/SendGrid 等の実接続・cron 配信・
   実送信は作らない。UIに「現在は設定の保存のみ」の注記を入れる。`emailVerified` も既定 false で土台のみ。

2. **共有文面に機密を入れない（型で防ぐ）**
   保存URL（originalUrl/affiliateUrl/productUrl）・匿名ID（userId）・非公開メモ（actualPriceMemo/note/
   priceMemo/参考リンク）を**シェア文面・共有URLに含めない**。`share.ts` の関数は機密フィールドを
   引数に取らないシグネチャにし、混入防止の回帰テストを必ず置く（設計 §7）。

3. **価格比較シェアはユーザー確認後のみ**
   価格比較シェアはワンタップ送信にしない。確認モーダルで文面プレビュー → 同意で `intent/tweet` を開く。
   セール情報シェアは公開情報のみだがプレビュー表示は推奨。

4. **広告は本番IDを入れない**
   `AdPlaceholder` は slot prop 対応にするが、AdSense の本番コード・本番IDは埋め込まない。
   破線枠＋「広告枠（未設定）」＋ slot 目印（`data-ad-slot`）まで。承認後に差し替える前提。

5. **後方互換（通知・記事・履歴）**
   旧 `NotificationSetting`（`email/timings` なし）を読んでも壊さない。読み出し時に
   `normalizeNotificationSetting` を必ず通す。`leadDays` は削除しない。記事は `excerpt/category/month`
   未設定でも一覧・metadata が成立すること（`getArticleExcerpt` でフォールバック）。

6. **リマインダーは純粋関数・ローカル算出**
   `generateReminderCandidates({now, sales, wishItems, setting})` は `now` を引数で受け、内部で
   `Date.now()` を呼ばない。EC突合は `merchantId` 直一致だけでなく `candidates[].merchantId` も対象
   （v1.1の候補配列を活用）。過去 `notifyAt` は除外。表示は件数とタイトルのみ（保存URL/ID/メモを出さない）。

7. **canonical / description**
   description に本文全文を流し込まない（記事は `excerpt` を使う）。各ページに `alternates.canonical` を
   相対パスで付与。セール詳細は `generateMetadata` が未実装なので追加する。

8. **法的ページは事実ベース・架空情報を断定しない**
   privacy/terms/about/contact を新設。運営者名・本番ドメイン・連絡先は環境に応じて埋める前提で、
   架空の会社名・住所を断定的に書かない。価格は手入力メモであり保証しない旨を明記。

9. **ロジックはユーティリティに切り出してテスト**
   excerpt 切詰め / 通知正規化 / メール検証 / リマインダー生成 / シェア文面 / 履歴 upsert を UI に
   ベタ書きしない。純粋関数化して単体テスト必須（設計 §11）。

10. **画像方針据え置き**
    商品画像・OG画像はプレースホルダー（自作SVG）のみ。外部画像の取得・表示はしない。

- 今回の回で動いた範囲（記事 / 広告 / 通知 / リマインダー / シェア / 履歴 / 法的）を列挙させる。

- 禁止事項チェックを自己申告させる:

  - 共有文面・共有URLに保存URL / 匿名ID / 非公開メモを含めていない。
