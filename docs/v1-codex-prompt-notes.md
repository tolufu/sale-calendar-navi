## 2. 既存プロンプトからの改善点（差分）

1. **「新規作成」→「差分実装」への言い換え**
   旧プロンプトは雛形〜全画面を作らせる前提だった。現状は実装済みなので、
   「§7のT-xxを順に。既存コンポーネント・リポジトリ抽象を再利用し、新規ファイルは最小限」と指示する。

2. **タスクIDで依頼単位を固定する**
   「T-01 編集機能だけ」のように **1依頼=1タスクID** に絞る。複数混ぜると差分レビューが難しくなる。

3. **「壊さない確認」をゴールに含める**
   依頼ごとに「既存の lint/test/build が緑のまま」を完了条件に明記（リグレッション防止）。

4. **v1.1先行実装の扱いを明記**
   `actualPriceMemo`・詳細折りたたみ・AF変換は既にある。**消さない・前倒しで増やさない**。
   v1では「表示文言が断定/公式誤認でないか」だけ確認させる。

5. **enum禁止の徹底を具体化**
   「色分けは `merchant.colorToken` 駆動にし、amazon/rakuten のハードコードを残さない」と
   ファイル名（`SaleCalendar.tsx` の `merchantClass`）まで指定する。

6. **テストを“書かせる対象”を明示**
   「ロジックは必ずユーティリティに切り出してから単体テスト」（例 `pickTopSales`）。
   インラインのまま放置させない。
