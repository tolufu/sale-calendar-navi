# v2.5以降 EC拡張基盤 実装報告

最終更新: 2026-05-30

## 1. 実装範囲

v2.5以降のEC追加に備え、既存のAmazon・楽天の挙動を維持しながら、merchant feature flag、
Provider境界、CSV/ASPフィードの型、公開共有のプライバシー基盤を追加した。

実データの新規取得、本番CSV取り込み、未提携サービスのAPI接続、スクレイピングは実装していない。

## 2. merchantsマスタ

`Merchant` に以下を追加した。

- `type`: `marketplace | retailer | b2b | other`
- `supportsAffiliate`
- `supportsApi`
- `supportsPriceAutoFetch`
- `supportsSaleCalendar`
- `affiliateProvider`
- `placeholderImageType`
- `integrationStatus`: `available | manual-only | planned`
- `urlHosts`

`Yahoo!ショッピング` は将来対応例として `planned` で追加した。商品URLの手動保存だけを許可し、
API、価格自動取得、画像自動取得、アフィリエイト変換、セールカレンダー連携は無効にしている。

URL判定はコード内固定辞書ではなく `merchants.urlHosts` を参照する。

## 3. Provider境界と安全ガード

`src/lib/providers/types.ts` に以下のインターフェースを追加した。

- `ProductSearchProvider`
- `AffiliateLinkProvider`
- `SaleCalendarProvider`
- `ProductFeedProvider`

`searchMerchantProducts` は `supportsApi` と `supportsPriceAutoFetch` の両方が有効なmerchantだけを
実行対象にする。既存の楽天検索Routeもこのガードを通す。未対応ECはProviderを呼び出さない。

楽天アフィリエイト変換は `affiliateLinkProviders` レジストリ経由に整理した。

## 4. CSV/ASPフィードの土台

`src/lib/import/csv/` に許可列、正規化型、純粋バリデーション関数を追加した。

- CSVインジェクションになり得る先頭文字を無害化する。
- 商品URL、アフィリエイトURL、画像URLは `https` のみ許可する。
- 画像URLは呼び出し側が指定する許可ホストだけを受け付ける。
- 未連携EC、無効merchant、必須列不足を拒否する。
- サンプルCSVは `docs/samples/product-feed-sample.csv` に置いた。

本番アップロード、永続化、管理画面、外部ASP接続は未実装。

## 5. 公開共有のプライバシー基盤

`createDefaultPublicShareSetting()` は `enabled: false` を返す。共有スナップショットはユーザーが
明示的に有効化した場合だけ生成する。

公開項目はホワイトリスト射影し、保存URL、匿名ID、内部ID、自由メモ、実質価格メモ、アフィリエイトURLを
含めない。タイトルにURL、匿名ID、`非公開メモ` が混入した場合も除去する。

公開Route、共有設定UI、永続化、noindexページは今後の実装対象。

## 6. 環境変数

`.env.example` に将来用トグル名だけを追加した。秘密情報は追加していない。

```bash
ENABLE_ASP_FEED=
ENABLE_CSV_IMPORT=
NEXT_PUBLIC_ENABLE_PUBLIC_SHARE=
```

## 7. テスト

以下を単体テストで確認する。

- merchant feature flag
- 未対応ECでは自動取得Providerを呼ばない
- CSV行の正常系、未連携EC拒否、許可外画像ホスト拒否、CSVインジェクション無害化
- 公開共有がデフォルト非公開
- 公開共有ペイロードに保存URL、匿名ID、内部ID、非公開メモを含めない

## 8. 未対応

- 未提携サービスのAPI、価格、画像、レビュー、在庫の取得
- スクレイピング
- CSV/ASPフィードの本番取り込み
- 公開共有ページと公開設定UI
- B2B画面

これらは提携、規約、認可、公開ページのレビュー後に別タスクで実装する。
