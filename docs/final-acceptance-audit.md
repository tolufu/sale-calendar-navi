## 2. PARTIAL / 要対応の詳細

### AC-010（⚠️ PARTIAL）v1〜v1.2 の画像ポリシー
- 現状: `WishlistForm.tsx` / `WishlistList.tsx` は楽天API候補の `imageSource: "rakuten_api"` のとき `<img src={candidate.imageUrl}>` を出典明記つきで表示する。自由な画像URL入力欄は存在しない（この点はAC-010を満たす）。
- 問題: 実装はv2.5だが、`CLAUDE.md`（「v1〜v1.2 の商品画像はプレースホルダーのみ」「外部の商品画像を取得・表示しない」）と AC-010 の文面が v1〜v1.2 前提のまま。v2.5コードと**ドキュメントが不整合**。
- 対象ファイル: `CLAUDE.md` / `docs/acceptance-checklist.md` / `src/features/wishlist/WishlistForm.tsx` / `src/features/wishlist/WishlistList.tsx`
- 必要対応（ドキュメント側、公開前）: v2以降は「楽天公式APIが返した画像URLのみ・出典明記・ユーザー任意画像URL入力は不可」というポリシーへ AC-010 とCLAUDE.mdを更新し、バージョン適用範囲を明示する。コード変更は不要。

### AC-012（⚠️ PARTIAL）AdSense申請できる状態
- 構造要件は充足: 記事5本（`src/data/articles.ts`）、`/privacy`・`/operator`・`/terms`、`/contact` 導線（フッター/ヘッダー）。
- 不足: 内容が公開可能水準でない。
  - `src/app/operator/page.tsx`: 運営者が「運営準備中」の仮情報。
  - `src/app/contact/page.tsx`: 送信ボタンが「送信準備中」で非機能。
  - `src/app/privacy/page.tsx` / `terms/page.tsx`: 「公開前の仮情報」表記。
  - 記事本文が各3段落程度で薄め。
- 必要対応（公開前）: 実運営者情報、機能する問い合わせ送信（または受付メール明記）、各記事のボリューム増、仮表記の除去。AdSense審査は薄いコンテンツ・連絡先不備で却下されやすい。

### AC-016（✅ PASS／注意）Firestore データ構造
- スキーマ・マスタは merchantId 方式で、`yahoo-shopping` を3社目として追加済み（enum固定でない）。拡張耐性あり＝AC合致。
- 注意: `src/lib/repositories/firestore.ts` の `createFirestoreRepositories()` は中身がローカルリポジトリのまま（Firestore永続化は未実装）。データ「構造」はAC合致だが、実際のFirestore接続は今後の実装が必要。公開後改善で可。
