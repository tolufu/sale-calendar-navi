# Claude Code 用プロンプト: v2.5 UI 修正

以下を Claude Code へ貼り付けて使用する。

```text
あなたは熟練したフルスタックエンジニア兼 UI/UX 実装担当です。

対象リポジトリ:
~/dev/workspaces/personal/sale-calendar-navi

最初に AGENTS.md を読み、必ず日本語で作業報告してください。
次に docs/handoffs/v2.5-ui-remediation-handoff.md を読み、そこに記載された
v2.5 UI 修正 PR を実装してください。

重要:
- docs/ui-reference/README.md と docs/ui-reference/01_*.png 〜 07_*.png を
  画面ごとのレビュー基準として必ず確認してください。
- UI 修正対象は v2.5 相当です。過去の v1 向けメモだけを根拠に実装しないでください。
- カレンダー初期表示月は固定値ではなく現在月にしてください。
- Amazon、楽天、Yahoo を UI に表示してください。
- 復旧機能は復旧コード方式です。ただし今回の PR では画面と導線のみを整備し、
  Firestore 未実装のため実処理が未提供であることを明示してください。
- Firestore の実保存、復旧コードによる実復旧、通知メールの実配信は別 PR です。
- merchantId / merchants マスタ方式、楽天公式 API 画像ポリシー、買い時スコア、
  価格計算、履歴、通知設定 UI、LocalStorage 互換性を維持してください。
- スクレイピング、任意画像 URL 入力、外部 EC 画像の無断取得、公式ロゴ、
  「最安値」「最安保証」等の断定表現を追加しないでください。
- docs/ui-reference/ の画像はレビュー専用です。public/ へコピーしないでください。
- `.env.local` や秘密情報をコミットしないでください。
- 既存コードを無断で全面上書きせず、共通 UI から各画面へ段階的に修正してください。

着手時に以下を実行して状態を報告してください。
pwd
git status --short
git branch --show-current
node -v
npm -v

実装順序:
1. 共通テーマ、Button、Card、ヘッダー、フッター、モバイルナビ
2. トップ、カレンダー、セール詳細
3. 欲しいもの登録、欲しいもの一覧
4. 履歴、記事詳細、必要なローディング・エラー・空状態
5. 復旧コード画面
6. 楽天画像 URL の保存時・描画時の再検証
7. テスト、ビルド、PC 幅約 1200px とスマートフォン幅約 390px の表示確認

実装後に以下を実行してください。
npm run lint
npm run test
npm run test:e2e
npm run build

最後に日本語で以下を報告してください。
1. 変更概要
2. 画面別の修正内容
3. 参考 UI と意図的に異なる点
4. 変更ファイル
5. 実行コマンド
6. 確認結果
7. 未対応・次にやること

不明点や資料間の矛盾がある場合は勝手に補完せず、実装前に具体的に報告してください。
```
